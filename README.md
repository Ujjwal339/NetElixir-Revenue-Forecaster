## Overview

A production-ready probabilistic forecasting utility that predicts future e-commerce **revenue and ROAS** across Google Ads, Microsoft (Bing) Ads, and Meta Ads.

Instead of a single optimistic number, the system outputs **confidence ranges (P10 / P50 / P90)** across 30, 60, and 90-day horizons вҖ” giving marketing agencies honest uncertainty bounds for budget planning.

AI-assisted causal summaries are generated via **Gemini 2.5 Flash**, with an automatic rule-based fallback when no API key is available.

## How the Pipeline Works

```
run.sh
  в”Ӯ
  в”ңв”Җ generate_features.py  (inference mode)
  в”Ӯ     вҖў Auto-detects Google / Bing / Meta CSVs by filename + schema
  в”Ӯ     вҖў Normalises Google cost micros вҶ’ USD
  в”Ӯ     вҖў Infers Meta campaign types from campaign names
  в”Ӯ     вҖў Builds 40 rolling features per campaign (see below)
  в”Ӯ     вҖў Outputs one feature row per campaign (latest date)
  в”Ӯ
  в””в”Җ predict.py
        вҖў Loads pickle/model.pkl (9 XGBoost quantile models)
        вҖў Generates P10 / P50 / P90 for every campaign Г— horizon
        вҖў Aggregates to: campaign вҶ’ campaign_type вҶ’ channel вҶ’ overall
        вҖў Writes predictions.csv
```

## Forecasting Methodology

### Model

**XGBoost with Quantile Loss** (`reg:quantileerror`)

- **9 models:** 3 horizons (30 / 60 / 90 days) Г— 3 quantile levels (P10 / P50 / P90)
- **Training data:** 20,279 (campaign, date) feature rows from 25,562 historical rows across 136 campaigns
- **Train/val split:** time-based 80/20 to prevent data leakage

### Validation Metrics

| Horizon | Quantile | MAE | RMSE | Pinball Loss | Coverage |
|---------|----------|-----|------|--------------|----------|
| 30d | P50 | $5,867 | $11,418 | 2,934 | 56% |
| 60d | P50 | $15,269 | $28,857 | 7,635 | 42% |
| 90d | P50 | $23,340 | $40,337 | 11,670 | 37% |

### Feature Engineering (40 Features)

| Group | Features |
|-------|----------|
| Rolling Spend | `spend_7d`, `spend_14d`, `spend_30d`, `spend_trend`, `spend_std_30` |
| Rolling Revenue | `revenue_7d`, `revenue_14d`, `revenue_30d`, `revenue_trend`, `revenue_std_30` |
| ROAS | `roas_7d`, `roas_14d`, `roas_30d`, `roas_trend`, `roas_std_30` |
| Engagement | `ctr_30d`, `cvr_30d`, `cpc_30d`, `cpm_30d`, `rpc_30d`, `aov_30d` |
| Budget | `budget_avg_30d`, `budget_util_30d`, `future_budget` |
| Lag Features | `revenue_lag1`, `revenue_lag7`, `spend_lag1`, `spend_lag7`, `roas_lag7` |
| Campaign | `campaign_age`, `campaign_efficiency` |
| Calendar | `day_of_week`, `day_of_month`, `week_of_year`, `month`, `quarter`, `is_weekend`, `is_holiday` |
| Categorical | `channel_enc`, `camp_type_enc` |

### Why Probabilistic?

Single-point forecasts are misleading for marketing budget decisions. Quantile regression gives honest bounds:

- **P10** вҖ” conservative plan; budget should not go below this
- **P50** вҖ” median expectation; primary forecast number
- **P90** вҖ” optimistic plan; upside if conditions are favorable

## Output Format (`predictions.csv`)

| Column | Description |
|--------|-------------|
| `forecast_date` | Date the forecast was generated |
| `aggregation_level` | `overall`, `channel`, `campaign_type`, or `campaign` |
| `channel` | `Google`, `Bing`, `Meta`, or `ALL` |
| `campaign_type` | e.g. `Search`, `PerformanceMax`, `Prospecting`, `ALL` |
| `campaign_name` | Campaign name or `ALL` |
| `period_days` | 30, 60, or 90 |
| `budget_input` | Projected spend over the period (USD) |
| `revenue_p10` | Conservative revenue estimate (USD) |
| `revenue_p50` | Median revenue estimate (USD) |
| `revenue_p90` | Optimistic revenue estimate (USD) |
| `roas_p10` | `revenue_p10 / budget_input` |
| `roas_p50` | `revenue_p50 / budget_input` |
| `roas_p90` | `revenue_p90 / budget_input` |

## AI Integration

The application combines **machine learning** with **generative AI** to provide actionable business insights while remaining fully functional without external AI services.

### Executive Forecast Summary

The dashboard optionally integrates **Gemini 2.5 Flash** to generate executive-friendly explanations of forecast results, including:

- Revenue outlook and expected ROAS
- Historical vs forecast performance comparison
- Forecast confidence and uncertainty analysis
- Seasonal trends and business risks
- Budget allocation recommendations
- Strategic marketing insights

If a Gemini API key is not provided, the application automatically switches to a **deterministic rule-based insight engine**, ensuring the dashboard remains fully operational.

### Historical Anomaly Detection

Historical campaign anomalies are detected using an **Isolation Forest** model based on:

- Campaign Spend
- Revenue
- ROAS

Detected anomalies are presented as structured business reports highlighting:

- Date and affected marketing channel
- Spend, Revenue, and ROAS
- Performance observations
- Possible business causes
- Recommended follow-up actions

Unlike the executive summary, anomaly detection is **fully deterministic** and does **not** require an LLM, ensuring consistent and reproducible results.

### Gemini API (Optional)

To enable AI-generated executive summaries, provide a **Gemini API Key** either:

- Through the Streamlit sidebar, or
- By setting the `GEMINI_API_KEY` environment variable.

Without an API key, the application automatically falls back to its built-in analytics engine with no loss of forecasting functionality.

Get a free Gemini API key:

https://aistudio.google.com/apikey

## Assumptions & Limitations

- Existing platform attribution is treated as source of truth (no re-attribution)
- Meta `conversion` column is interpreted as revenue value in USD
- Forecasts are aggregate-period totals, not daily
- Model generalises across campaigns using statistical features, not campaign IDs вҖ” handles unseen campaigns at test time
- Forecast uncertainty increases beyond 60 days due to seasonal compounding
- MAPE is unreliable for this dataset due to zero-revenue days; MAE and Pinball Loss are the primary metrics
