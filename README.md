## Overview

A production-ready probabilistic forecasting utility that predicts future e-commerce **revenue and ROAS** across Google Ads, Microsoft (Bing) Ads, and Meta Ads.

Instead of a single optimistic number, the system outputs **confidence ranges (P10 / P50 / P90)** across 30, 60, and 90-day horizons — giving marketing agencies honest uncertainty bounds for budget planning.

AI-assisted causal summaries are generated via **Gemini 2.5 Flash**, with an automatic rule-based fallback when no API key is available.

## Quickstart

**Python 3.12+**

```bash
git clone https://github.com/Ujjwal339/Netelixir-Revenue-Forecaster.git
cd Netelixir-Revenue-Forecaster
pip install -r requirements.txt
```



```bash
streamlit run app/streamlit_app.py
```

Opens at `http://localhost:8501`

## How the Pipeline Works

```
run.sh
  │
  ├─ generate_features.py  (inference mode)
  │     • Auto-detects Google / Bing / Meta CSVs by filename + schema
  │     • Normalises Google cost micros → USD
  │     • Infers Meta campaign types from campaign names
  │     • Builds 40 rolling features per campaign (see below)
  │     • Outputs one feature row per campaign (latest date)
  │
  └─ predict.py
        • Loads pickle/model.pkl (9 XGBoost quantile models)
        • Generates P10 / P50 / P90 for every campaign × horizon
        • Aggregates to: campaign → campaign_type → channel → overall
        • Writes predictions.csv
```

## Forecasting Methodology

### Model

**XGBoost with Quantile Loss** (`reg:quantileerror`)

- **9 models:** 3 horizons (30 / 60 / 90 days) × 3 quantile levels (P10 / P50 / P90)
- **Training data:** 20,279 (campaign, date) feature rows from 25,562 historical rows across 136 campaigns
- **Train/val split:** time-based 80/20 to prevent data leakage

### Validation Metrics

| Horizon | Quantile | MAE | RMSE | Pinball Loss | Coverage |
|---------|----------|-----|------|--------------|----------|
| 30d | P50 | $5,867 | $11,418 | 2,934 | 56% |
| 60d | P50 | $15,269 | $28,857 | 7,635 | 42% |
| 90d | P50 | $23,340 | $40,337 | 11,670 | 37% |

### Feature Engineering (40 Features)

| Group | Features |
|-------|----------|
| Rolling Spend | `spend_7d`, `spend_14d`, `spend_30d`, `spend_trend`, `spend_std_30` |
| Rolling Revenue | `revenue_7d`, `revenue_14d`, `revenue_30d`, `revenue_trend`, `revenue_std_30` |
| ROAS | `roas_7d`, `roas_14d`, `roas_30d`, `roas_trend`, `roas_std_30` |
| Engagement | `ctr_30d`, `cvr_30d`, `cpc_30d`, `cpm_30d`, `rpc_30d`, `aov_30d` |
| Budget | `budget_avg_30d`, `budget_util_30d`, `future_budget` |
| Lag Features | `revenue_lag1`, `revenue_lag7`, `spend_lag1`, `spend_lag7`, `roas_lag7` |
| Campaign | `campaign_age`, `campaign_efficiency` |
| Calendar | `day_of_week`, `day_of_month`, `week_of_year`, `month`, `quarter`, `is_weekend`, `is_holiday` |
| Categorical | `channel_enc`, `camp_type_enc` |

### Why Probabilistic?

Single-point forecasts are misleading for marketing budget decisions. Quantile regression gives honest bounds:

- **P10** — conservative plan; budget should not go below this
- **P50** — median expectation; primary forecast number
- **P90** — optimistic plan; upside if conditions are favorable

## Output Format (`predictions.csv`)

| Column | Description |
|--------|-------------|
| `forecast_date` | Date the forecast was generated |
| `aggregation_level` | `overall`, `channel`, `campaign_type`, or `campaign` |
| `channel` | `Google`, `Bing`, `Meta`, or `ALL` |
| `campaign_type` | e.g. `Search`, `PerformanceMax`, `Prospecting`, `ALL` |
| `campaign_name` | Campaign name or `ALL` |
| `period_days` | 30, 60, or 90 |
| `budget_input` | Projected spend over the period (USD) |
| `revenue_p10` | Conservative revenue estimate (USD) |
| `revenue_p50` | Median revenue estimate (USD) |
| `revenue_p90` | Optimistic revenue estimate (USD) |
| `roas_p10` | `revenue_p10 / budget_input` |
| `roas_p50` | `revenue_p50 / budget_input` |
| `roas_p90` | `revenue_p90 / budget_input` |

## AI Integration

`llm_insights.py` integrates **Gemini 2.5 Flash** (free tier) to generate:

- **Executive forecast summaries** — explains why revenue is expected to rise or fall
- **Anomaly detection** — uses Isolation Forest to find statistical outliers in historical data, then generates LLM narratives for each anomaly
- **Risk identification** — budget pacing, attribution lag, seasonality warnings
- **Budget allocation recommendations** — channel-level rebalancing advice

Set `GEMINI_API_KEY` in your environment or paste it into the Streamlit sidebar.
Falls back to deterministic rule-based summaries with no API key required.

Get a free key: https://aistudio.google.com/apikey

## Assumptions & Limitations

- Existing platform attribution is treated as source of truth (no re-attribution)
- Meta `conversion` column is interpreted as revenue value in USD
- Forecasts are aggregate-period totals, not daily
- Model generalises across campaigns using statistical features, not campaign IDs — handles unseen campaigns at test time
- Forecast uncertainty increases beyond 60 days due to seasonal compounding
- MAPE is unreliable for this dataset due to zero-revenue days; MAE and Pinball Loss are the primary metrics
